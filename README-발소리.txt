[발소리 (Footsteps)] dot-games-host · 촉각+공간음향+TTS 삼중 채널 서바이벌  2026-07-23

■ 게임: 숨바꼭질 서바이벌 (6x5 타일, 60x40 핀에 10x8 블록씩 매핑)
  괴물을 피해 정해진 라운드(또는 연습 모드)까지 버티는 턴제 스텔스 게임.
  방향키로 움직이면 소리가 나 위치가 들키고(3턴간 집중 추적), 가만히 있으면 안전.
  덤불 타일 3곳에 숨으면 더 안전해지지만 내 청각도 함께 먹먹해지는 트레이드오프.

■ 접근성 우선 UX (눈 없이 바로 이해·플레이)
  · 시작 시 "숨을 죽이세요. 발소리가 들리기 시작해요." 안내, 규칙 듣기 버튼으로 언제든 전체 설명 재생.
  · 매 턴마다 상황을 음성으로: "숨소리가 들려요… 바로 옆이에요." / "가만히 있어… 지나갔어요." /
    "지금이 기회예요! 덤불로 뛰어들 수 있어요." 등 거리·이벤트 기반 내레이션.
  · 공간 음향: 몬스터 발소리는 좌우 스테레오 팬(방향)+음량/피치(거리)로, 1칸 이내 접근 시 숨소리 루프 추가.
    내가 낸 발소리는 다른 음색(노이즈 틱)이라 몬스터 소리와 혼동되지 않음.
  · DotPad 촉각: 내 위치=항상 꽉 찬 블록(기준점), 덤불=속이 빈 테두리(지형지물), 몬스터=이동 순간 잠깐 꽉 찬 블록(발소리를 "느낌").
  · 5라운드마다 "N라운드째예요" 진행 안내, 종료 시 생존 라운드·최고 기록 갱신 여부까지 TTS로 명시.
  · 일시정지/결과 창은 role="dialog"+aria-modal, 여는 순간 포커스 이동·Tab 포커스 트랩·닫을 때 포커스 복원까지 처리.
  · 촉각 방향 버튼(F1~F4/화면 버튼) 전부 스크린리더 접근 가능(터치스크린 단독 사용자도 조작 가능), 난이도 선택 버튼은 aria-pressed로 현재 선택 상태 인지 가능.

■ 조작 요약
  키보드: 방향키 이동 · S 가만히 있기 · R 안내 다시 듣기 · P 일시정지 · H 도움말.
  DotPad: F1~F4 상/하/좌/우 이동 · PanningAll 가만히 있기 · PanningLeft 안내 다시 듣기 ·
          PanningRight 도움말 · LPF1 일시정지.
    (물리 DotPad엔 방향키가 없어, 닷 미로와 동일하게 F1~F4를 방향으로 사용)
  승/패 순간엔 displayAllUp/Down 몇 회 깜빡임으로 촉각 펄스도 함께 전달.
  소리 켜기/끄기: 다른 5개 게임과 동일한 스피커 아이콘 토글(화면 상시 노출). 꺼도 자막은 유지되고
    TTS·효과음만 함께 음소거(다른 게임들의 soundOn 패턴 그대로 재사용).

■ 배경 이미지: assets/bg.webp (제공된 footsteps-gameplay 게임플레이 아트: 야간 숲길 + 발광 발자국 트레일).
  linear-gradient 오버레이 + cover 방식으로 기존 배경 규약 그대로 적용.

■ 구조(외부 자산 표준): dotfootsteps/ = index.html + assets/(bg.webp) + dotpad-sdk/(DotPadSDK-3.0.0.js)
  tts.js는 호스트 루트 공용 파일을 그대로 참조(<script src="/tts.js">), 별도 번들 없음.

■ 배포: dotfootsteps/ 폴더째 업로드(폴더명 반드시 'dotfootsteps') → 재배포 → 하드 새로고침.
  확인: dot-games-host.vercel.app/dotfootsteps/index.html?embed=1

■ 인코딩/불변식: dotBit=lx*4+ly, 2x4 셀, 60x40 → 300바이트/600hex. 닷 미로의 EA·encodeFrame과 동일 결과.

■ 알려진 제약: 실제 DotPadSDK-3.0.0.js는 동적 import(ES 모듈)로 로드되므로, 이 파일을 지원하지 않는
  구형 환경(또는 이 zip 검토에 쓰인 jsdom 테스트 환경)에서는 하드웨어 연결 없이 조용히 폴백됩니다.
  실제 브라우저(Chrome/Edge)+실기기에서는 닷 미로와 동일한 방식으로 정상 동작합니다.
